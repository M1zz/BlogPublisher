# 기술 블로그를 시작하게 된 이유

오늘 처음으로 블로그 글을 쓴다. 사실 블로그를 시작해야겠다고 마음먹은 건 꽤 됐는데, 계속 미루다가 이제야 첫 글을 쓴다.

---

## 계기

솔직히 말하면 시작은 별거 아니었다. 어느 날 회사에서 코드 리뷰를 받다가 시니어 개발자분이 "이거 왜 이렇게 짰어요?"라고 물었는데, 나는 대답을 못했다. 그냥 돌아가니까 그렇게 짠 거였다. 그때 뭔가 부끄러웠다. 내가 짠 코드인데 설명을 못하다니.

그날 이후로 뭔가 바뀌어야겠다고 생각했다. 근데 뭘 어떻게 해야 할지 모르겠더라. 책을 읽어도 금방 까먹고, 강의를 들어도 실제로 쓸 일이 없으면 머릿속에서 사라졌다.

---

## 왜 블로그인가

처음엔 노션에 정리하려고 했다. 근데 노션은 나만 보니까 대충 적게 되더라. "나중에 다시 정리해야지" 하면서 두서없이 메모만 늘어나고. 결국 다시 안 보게 됐다.

그러다가 어떤 블로그 글을 읽었는데, 그 사람이 쓴 글이 엄청 명쾌했다. 나도 똑같이 삽질했던 내용인데, 그 사람은 왜 그런 문제가 생기는지, 어떻게 해결하는지 깔끔하게 정리해뒀더라. 그때 깨달았다. 아, 남한테 설명하려고 쓰면 대충 못 쓰는구나.

---

## 플랫폼 선택의 늪

블로그 쓰겠다고 마음먹고 나서 플랫폼 고르는 데만 2주를 썼다. 진짜 별거 아닌 것 같은데 찾아보면 찾아볼수록 뭘 골라야 할지 모르겠더라.

### 후보들

처음에 고려했던 플랫폼들이다.

**티스토리**
국내에서 가장 많이 쓰는 것 같아서 제일 먼저 봤다. 네이버 검색에 잘 노출되고, 한글 커뮤니티도 크고. 근데 카카오 계정이 필요하고, 초대장 시스템이 좀 번거롭더라. 그리고 디자인 커스터마이징하려면 HTML/CSS 직접 건드려야 해서 귀찮았다.

**벨로그**
개발자들 사이에서 요즘 많이 쓰는 것 같아서 봤다. UI가 깔끔하고 마크다운 지원도 잘 되어 있더라. 근데 커스텀 도메인을 연결할 수가 없었다. 나중에 내 도메인으로 운영하고 싶었는데, 그게 안 되니까 좀 아쉬웠다.

**미디엄**
영어권에서 유명한 플랫폼. 디자인이 예쁘고 읽기 좋은 건 맞는데, 한글 글은 노출이 거의 안 되는 것 같았다. 그리고 무료 글 읽기 제한이 있어서 독자 입장에서 불편할 것 같았다.

**깃허브 페이지 + Jekyll**
직접 세팅하면 완전 자유롭게 커스터마이징할 수 있어서 끌렸다. 근데 세팅하는 데 시간이 너무 많이 들 것 같았다. 블로그 글 쓰려고 시작한 건데 세팅만 하다가 지칠 것 같아서 일단 패스.

**DEV.to**
해외 개발자 커뮤니티. 글 올리면 바로 개발자들한테 노출되는 게 장점이었다. 근데 여기도 커스텀 도메인 연결이 안 됐다.

### 내가 원했던 것들

플랫폼 비교하면서 내가 뭘 원하는지 정리가 됐다.

1. **커스텀 도메인 지원** - 내 도메인으로 블로그 운영하고 싶었다. 나중에 플랫폼 바꾸더라도 URL이 유지되면 좋겠다는 생각.

2. **검색 노출** - 글 써놓고 아무도 안 보면 의미가 없으니까. 구글 검색에 잘 잡혀야 했다.

3. **마크다운 지원** - 코드 블록이랑 문법 하이라이팅이 잘 되어야 했다. 개발 블로그니까.

4. **세팅이 간단할 것** - 블로그 세팅하느라 시간 쓰기 싫었다. 그냥 글 쓰는 데 집중하고 싶었다.

5. **무료** - 취미로 시작하는 건데 돈 내기 싫었다.

## 결국 Hashnode

이것저것 비교하다가 Hashnode로 정했다. 솔직히 처음엔 이름도 몰랐는데, 검색하다가 발견했다.

### 선택한 이유들

**무료 커스텀 도메인**
이게 제일 컸다. 다른 플랫폼들은 커스텀 도메인 쓰려면 유료 플랜 써야 하는데, Hashnode는 무료로 된다. 내 도메인 연결하면 `blog.mydomain.com` 이런 식으로 운영할 수 있다. SSL 인증서도 자동으로 붙여준다.

**구글 검색 노출이 잘 됨**
이건 직접 써보고 알게 된 건데, 글 올리고 며칠 지나니까 구글에서 검색이 됐다. SEO 설정도 글마다 할 수 있어서 메타 태그 같은 거 신경 쓸 수 있다. 사이트맵도 자동으로 생성해준다.

**마크다운 에디터가 좋음**
글 쓸 때 마크다운으로 쓰면 실시간으로 프리뷰가 나온다. 코드 블록 문법 하이라이팅도 언어별로 잘 된다. 이미지 업로드도 드래그앤드롭으로 되고, Hashnode CDN에 올라가서 따로 이미지 호스팅 걱정 안 해도 된다.

**개발자 커뮤니티**
Hashnode 자체가 개발자 블로그 플랫폼이라서, 글 올리면 Hashnode 피드에도 노출된다. 운 좋으면 다른 개발자들한테 읽힐 수도 있다. 물론 영어 글이 대부분이라 한글 글이 엄청 노출되진 않겠지만.

**GitHub 연동**
글을 GitHub 저장소에 백업할 수 있다. 블로그 플랫폼이 망하거나 정책이 바뀌어도 내 글은 내 저장소에 남아있으니까 안심이 된다.

**세팅이 진짜 간단함**
가입하고 블로그 이름 정하면 끝이다. 테마도 몇 개 있어서 그냥 고르면 되고, 굳이 커스터마이징 안 해도 기본 디자인이 깔끔하다.

### 단점도 있다

솔직히 단점도 있다.

- 한글 사용자가 거의 없어서 한글 관련 정보 찾기가 힘들다
- 네이버 검색에는 잘 안 잡히는 것 같다 (구글은 잘 됨)
- 커뮤니티 기능이 영어 중심이라 한국어 글은 노출이 적다

근데 나는 구글 검색 노출이랑 커스텀 도메인이 더 중요해서 이 정도 단점은 감수하기로 했다.

## 사실 무섭다

근데 막상 공개 블로그를 시작하려니까 무섭다. 내가 쓴 글에 틀린 내용이 있으면 어쩌지? 누가 "이거 틀렸는데요"라고 댓글 달면 창피하잖아.

그래도 시작해보려고 한다. 틀리면 고치면 되니까. 오히려 누가 틀린 거 알려주면 배우는 거 아닌가. 맞는 것만 쓰려고 하면 아무것도 못 쓸 것 같다.

---

## 앞으로 어떤 글을 쓸까

거창한 건 안 쓸 거다. 그냥 내가 삽질한 거, 새로 알게 된 거, 헷갈렸는데 정리하면서 이해된 거. 그런 것들.

일단 Swift랑 iOS 개발 관련 글을 주로 쓸 것 같다. 요즘 SwiftUI 배우면서 삽질한 거 정리해둘 게 많다.

나중에 내가 다시 봤을 때 도움이 되면 좋겠고, 운 좋으면 나처럼 헤매는 누군가한테도 도움이 될 수도 있겠지.

일단 오늘은 여기까지. 첫 글이라 뭘 써야 할지 몰라서 그냥 생각나는 대로 썼다.
